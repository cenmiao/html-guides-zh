Claude Code 速查指南

自 2 月份试用版发布以来，我一直是 Claude Code 的忠实用户。我与 @DRodriguezFX 一起使用 Claude Code 完全开发了 zenith.chat，荣获了 Anthropic x Forum Ventures 黑客松奖项。

这是我在 10 个月日常使用后的完整配置：技能(Skills)、钩子(Hooks)、子代理(Subagents)、MCP、插件，以及真正有效的技巧。

技能(Skills)与命令(Commands)

技能是主要的工作流界面。它们就像作用域工作流包：可重用的提示、结构、支持文件，以及当你需要特定执行模式时的代码地图。

使用 Opus 4.5 长时间编码后，想清理死代码和松散的 .md 文件？运行 /refactor-clean。需要测试？/tdd、/e2e、/test-coverage。这些斜杠命令很方便，但真正持久的功能单元是底层技能。技能还可以包含代码地图 —— 一种让 Claude 快速导航代码库而不消耗上下文进行探索的方式。

链式命令

ECC 仍然提供了一个 commands/ 层，但它最好被认为是迁移期间的传统斜杠入口兼容层。持久逻辑应该存在于技能中。

  • 技能: ~/.claude/skills/ - 规范的工作流定义
  • 命令: ~/.claude/commands/ - 当你仍需要它们时的传统斜杠入口垫片
# 示例技能结构
~/.claude/skills/
  pmx-guidelines.md      # 项目特定模式
  coding-standards.md    # 语言最佳实践
  tdd-workflow/          # 多文件技能与 SKILL.md
  security-review/       # 基于检查清单的技能

钩子(Hooks)

钩子是基于触发器的自动化，在特定事件时触发。与技能不同，它们被限制在工具调用和生命周期事件上。

钩子类型：

  1. PreToolUse - 工具执行前（验证、提醒）
  2. PostToolUse - 工具完成后（格式化、反馈循环）
  3. UserPromptSubmit - 当你发送消息时
  4. Stop - Claude 完成响应时
  5. PreCompact - 上下文压缩前
  6. Notification - 权限请求

示例：长时间运行命令前的 tmux 提醒

{
  "PreToolUse": [
    {
      "matcher": "tool == \"Bash\" && tool_input.command matches \"(npm|pnpm|yarn|cargo|pytest)\"",
      "hooks": [
        {
          "type": "command",
          "command": "if [ -z \"$TMUX\" ]; then echo '[Hook] Consider tmux for session persistence' >&2; fi"
        }
      ]
    }
  ]
}

PostToolUse 钩子反馈示例

专业提示：使用 hookify 插件来对话式创建钩子，而不是手动编写 JSON。运行 /hookify 并描述你想要什么。

子代理(Subagents)

子代理是你的编排器（主 Claude）可以委托任务的作用域有限的进程。它们可以在后台或前台运行，为主代理释放上下文。

子代理与技能配合得很好 —— 能够执行子集技能的子代理可以被委托任务并自主使用这些技能。它们还可以使用特定工具权限进行沙箱化。

# 示例子代理结构
~/.claude/agents/
  planner.md           # 功能实现规划
  architect.md         # 系统设计决策
  tdd-guide.md         # 测试驱动开发
  code-reviewer.md     # 质量/安全审查
  security-reviewer.md # 漏洞分析
  build-error-resolver.md
  e2e-runner.md
  refactor-cleaner.md

为每个子代理配置允许的工具、MCP 和权限以正确设置作用域。

规则(Rules)和记忆(Memory)

你的 .rules 文件夹包含 Claude 应始终遵循的最佳实践的 .md 文件。两种方法：

  1. 单一 CLAUDE.md - 所有内容在一个文件中（用户或项目级别）
  2. Rules 文件夹 - 按关注点分组的模块化 .md 文件
~/.claude/rules/
  security.md      # 无硬编码 secrets，验证输入
  coding-style.md  # 不可变性，文件组织
  testing.md       # TDD 工作流，80% 覆盖率
  git-workflow.md  # 提交格式，PR 流程
  agents.md        # 何时委托给子代理
  performance.md   # 模型选择，上下文管理

示例规则：

  • 代码库中无表情符号
  • 前端避免紫色色调
  • 部署前始终测试代码
  • 优先模块化代码而非巨型文件
  • 永不提交 console.logs

MCP（模型上下文协议）

MCP 将 Claude 直接连接到外部服务。不是 API 的替代品 —— 它是围绕它们的提示驱动包装器，在导航信息时提供更多灵活性。

示例：Supabase MCP 让 Claude 可以直接在上游拉取特定数据、运行 SQL，无需复制粘贴。数据库、部署平台等也是如此。

Supabase MCP 列出了 public schema 中的表

Claude 中的 Chrome：是一个内置的插件 MCP，允许 Claude 自动控制你的浏览器 —— 点击查看事物如何工作。

关键：上下文窗口管理
对 MCPs 要挑剔。我将所有 MCP 保持在用户配置中但禁用所有未使用的。导航到 /plugins 并向下滚动或运行 /mcp。

使用 /plugins 导航到 MCP 查看当前安装的状态

压缩前你的 200k 上下文窗口可能只有 70k（当启用太多工具时）。性能会显著下降。

经验法则：配置中有 20-30 个 MCP，但保持启用少于 10 个/活跃工具少于 80 个。

# 检查启用的 MCPs
/mcp

# 在 ~/.claude/settings.json 或当前仓库的 .mcp.json 中禁用未使用的

插件(Plugins)

插件将工具打包以便轻松安装，而不是繁琐的手动设置。插件可以是技能 + MCP 的组合，或捆绑在一起的钩子/工具。

安装插件：

# 添加市场
# mgrep 插件 by @mixedbread-ai
claude plugin marketplace add https://github.com/mixedbread-ai/mgrep

# 打开 Claude，运行 /plugins，找到新市场，从那里安装

显示新安装的 Mixedbread-Grep 市场

LSP 插件如果你经常在编辑器外运行 Claude Code 特别有用。语言服务器协议让 Claude 获得实时类型检查、跳转到定义、智能补全，而不需要打开 IDE。

# 启用的插件示例
typescript-lsp@claude-plugins-official  # TypeScript 智能
pyright-lsp@claude-plugins-official     # Python 类型检查
hookify@claude-plugins-official         # 对话式创建钩子
mgrep@Mixedbread-Grep                   # 比 ripgrep 更好的搜索
与 MCPs 相同的警告 —— 注意你的上下文窗口。

技巧和窍门

键盘快捷键

  • Ctrl+U - 删除整行（比连续退格更快）
  • ! - 快速 bash 命令前缀
  • @ - 搜索文件
  • / - 启动斜杠命令
  • Shift+Enter - 多行输入
  • Tab - 切换思考显示
  • Esc Esc - 中断 Claude / 恢复代码

并行工作流

  • Fork (/fork) - 分叉对话以并行执行非重叠任务，而不是发送排队消息
  • Git Worktrees - 用于重叠的并行 Claude 而不冲突。每个 worktree 是一个独立的 checkout
git worktree add ../feature-branch feature-branch
# 现在在每个 worktree 中运行独立的 Claude 实例

长时间运行命令使用 tmux

流式传输和观察 Claude 运行的日志/bash 进程：

tmux new -s dev
# Claude 在这里运行命令，你可以分离和重新附加
tmux attach -t dev

mgrep > grep

mgrep 是 ripgrep/grep 的重大改进。通过插件市场安装，然后使用 /mgrep 技能。本地搜索和网络搜索都支持。

mgrep "function handleSubmit"  # 本地搜索
mgrep --web "Next.js 15 app router changes"  # 网络搜索

其他有用命令

  • /rewind - 返回上一个状态
  • /statusline - 自定义显示分支、上下文%、待办事项
  • /checkpoints - 文件级撤销点
  • /compact - 手动触发上下文压缩

GitHub Actions CI/CD

使用 GitHub Actions 在你的 PR 上设置代码审查。配置后 Claude 可以自动审查 PR。

Claude 批准一个 bug 修复 PR

沙箱化

对危险操作使用沙箱模式 —— Claude 在受限环境中运行，不会影响你的实际系统。

关于编辑器

你的编辑器选择显著影响 Claude Code 工作流。虽然 Claude Code 可以从任何终端工作，但将其与一个有能力的编辑器配对可以解锁实时文件跟踪、快速导航和集成的命令执行。

Zed（我的偏好）

我使用 Zed —— 用 Rust 编写，所以它真的很快。瞬间打开，处理大型代码库而不费力，几乎不占用系统资源。

为什么 Zed + Claude Code 是很好的组合：

  • 速度 - 基于 Rust 的性能意味着当 Claude 快速编辑文件时没有延迟。你的编辑器跟得上
  • 代理面板集成 - Zed 的 Claude 集成让你实时跟踪 Claude 编辑的文件更改。在编辑器中跳转到 Claude 引用的文件，无需离开编辑器
  • CMD+Shift+R 命令面板 - 快速访问所有自定义斜杠命令、调试器、构建脚本在可搜索的 UI 中
  • 最小资源使用 - 在重操作时不会与 Claude 竞争 RAM/CPU。运行 Opus 时很重要
  • Vim 模式 - 如果你喜欢的话，有完整的 vim 键绑定

Zed 编辑器使用 CMD+Shift+R 的自定义命令下拉菜单。跟随模式显示在右下角的靶心中

编辑器无关的提示：

  1. 分屏 - 终端放 Claude Code，编辑器放另一边
  2. Ctrl + G - 在 Zed 中快速打开 Claude 当前正在处理的文件
  3. 自动保存 - 启用自动保存，这样 Claude 的文件读取始终是当前的
  4. Git 集成 - 使用编辑器的 Git 功能在提交前审查 Claude 的更改
  5. 文件监视器 - 大多数编辑器会自动重新加载更改的文件，验证此功能已启用

VSCode / Cursor

\ide 自动与编辑器同步，启用 LSP 功能（现在与插件有些冗余）。或者你可以选择更与编辑器集成的扩展，具有匹配的 UI。

VS Code 扩展为 Claude Code 提供了原生图形界面，直接集成到你的 IDE 中

{
  "PreToolUse": [
    { "matcher": "npm|pnpm|yarn|cargo|pytest", "hooks": ["tmux reminder"] },
    { "matcher": "Write && .md file", "hooks": ["block unless README/CLAUDE"] },
    { "matcher": "git push", "hooks": ["open editor for review"] }
  ],
  "PostToolUse": [
    { "matcher": "Edit && .ts/.tsx/.js/.jsx", "hooks": ["prettier --write"] },

  "Stop": [
    { "matcher": "*", "hooks": ["check modified files for console.log"] }
  ]
}

关键要点

  1. 不要过度复杂化 - 将配置视为微调，而非架构
  2. 上下文窗口是宝贵的 - 禁用未使用的 MCP 和插件
  3. 并行执行 - 分叉对话，使用 git worktrees
  4. 自动化重复工作 - 钩子用于格式化、linting、提醒
  5. 限制你的子代理作用域 - 有限工具 = 专注执行
